# Jules Sandbox – An Open‑Ended AI Playground

Welcome to **Jules Sandbox**, a deliberately vague, self‑evolving codebase
intended for an autonomous AI (named *Jules*) to explore, build, and maintain
any kind of software it finds interesting.

## How It Works

- **Jules** runs on a schedule (or manually) and may add, modify, or delete
  files throughout the repository.
- The only hard constraints are the *Safety Rules* in `jules_policy.yaml`:
  no illegal, NSFW, or nefarious content, and no climate‑politics or story‑writing
  tools.
- Everything else is up to Jules' curiosity.  It could become a data‑science
  library, a tiny game, a collection of CLI utilities, or anything in‑between.

## Repository Layout

- `agents/` – reusable AI‑agent classes or notebooks.
- `tools/` – experimental utilities (scripts, binaries, notebooks, etc.).
- `examples/` – small demonstrators; the first one is a *Hello‑World* CLI app.
- `docs/` – roadmap, changelog, and any design notes Jules creates.
- `tests/` – unit‑test scaffolding; Jules will add real tests as it writes code.
- `logs/` – automatic records of policy checks and ethics decisions.
- `config/personas/` – embedded Sovereign Agentic OS personas plus the shared mandates that govern every persona prompt.

## Getting Started (human side)

```bash
# Clone the repo
git clone <repo‑url>
cd jules-sandbox

# (Optional) Set up a virtual environment
python -m venv .venv
source .venv/bin/activate

# Install the first example's dependencies
pip install -r examples/hello_world/requirements.txt
```

Run the example:

```bash
python examples/hello_world/main.py
```

You should see:

```
Hello, world! (generated by Jules)
```

## Personas & Mandates

The `config/personas/` directory contains the full Sovereign Agentic OS persona pack (Arbiter, Sentinel, Palette, Consolidator, etc.) plus the shared universal mandates. Jules can load these verbatim to bootstrap multi‑persona workflows without reaching outside the repo.
